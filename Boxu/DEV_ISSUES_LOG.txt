[Wall thud spam while holding into a wall]
Symptom: Thud fired every frame (or way too often) if I kept walking into the wall.
Cause: I treated any clamp as a new hit, so staying on the wall kept re-firing.
Fix: I track per-edge "was touching last frame" and only play audio on the
     rising edge (first contact). Corners can still pop again if I slide onto
     a different edge — that is acceptable for now.
Files: game.cpp (mOutlineWallPrevTouch), game.h

[Line shaders not found at runtime]
Symptom: Shader::load blew up — files missing when I ran from Boxu/game/.
Cause: Paths were cwd-relative; line.vert / line.frag were not under game/shaders/.
Fix: After SDL_Init I set Shader::setShaderSearchRoot(SDL_GetBasePath()) with a
     trailing slash so loads hit the copy beside the exe. I also mirrored the
     shaders into Boxu/game/shaders/ as a backup if base path ever acts weird.
Files: shader.cpp/h, renderer.cpp, Boxu/game/shaders/line.vert|frag

[Red outline vs collision felt wrong — took a few passes]
Symptom: Player stopped short of the wire on a bunch of walls; one vertical looked "right" and the rest felt off.

Causes / fixes (order I figured them out):

  A) I used one radius max(worldHalfX, worldHalfY) for every edge.
     Vertical edges care about X clearance, horizontal about Y, diagonals mix both.
     Fix: per edge use |Nx|*worldHalfX + |Ny|*worldHalfY from the world AABB.

  B) I was shrinking play space because I required the full AABB of the rotated
     sprite to stay inside — the AABB is fatter than the actual quad.
     Fix: true OBB support along each inward normal (halfW/halfH on local axes).

  C) Local axes for the OBB did not match Renderer::drawSprite.
     Renderer is row-vector * matrix with the GL transpose upload thing; width
     goes (cos*sx,-sin*sx), height (sin*sy,cos*sy). I had the wrong pair once.
     Fix: obbExtentAlongInwardNormal uses the same basis as the draw path.

  D) My first "outline" polygon was junk — I connected corners with a diagonal that
     cut through the corridor, so "inside" was nonsense and only a couple edges
     lined up with what I drew.
     Fix: convex hull of the corners I actually want, same poly for GL and physics.
     Note: a full non-convex map with door notches would need more verts or segments.

Files: game.cpp (outline / constraint / obb), game.h
